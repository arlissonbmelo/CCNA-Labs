Objectives | Objetivos

**EN:**
- Design and implement a small-to-medium campus network topology.
- Apply VLAN segmentation across multiple access switches.
- Implement redundancy at Layer 2 and Layer 3.
- Provide dynamic IP addressing using DHCP.
- Simulate internet access using a loopback network.
- Document design decisions and configurations clearly.

**PT-BR:**
- Projetar e implementar uma topologia de rede de campus de pequeno a médio porte.
- Aplicar segmentação de rede utilizando VLANs em múltiplos switches de acesso.
- Implementar redundância nas camadas 2 e 3.
- Fornecer endereçamento IP dinâmico utilizando DHCP.
- Simular acesso à internet utilizando uma rede loopback.
- Documentar claramente as decisões de design e as configurações realizadas.

Network Architecture | Arquitetura da Rede

**EN:**  
The network follows a hierarchical campus design, divided into access, core, and edge layers.  
Access switches connect end devices and provide VLAN segmentation.  
Core switches handle inter-VLAN routing, redundancy, and high availability using HSRP.  
Edge routers provide connectivity to an external network, simulating internet access through a loopback address.

**PT-BR:**  
A rede segue um modelo hierárquico de campus, dividida em camadas de acesso, core e borda.  
Os switches de acesso conectam os dispositivos finais e realizam a segmentação por VLANs.  
Os switches de core são responsáveis pelo roteamento entre VLANs, redundância e alta disponibilidade utilizando HSRP.  
Os roteadores de borda fornecem conectividade com uma rede externa, simulando o acesso à internet através de um endereço loopback.

Technologies and Protocols | Tecnologias e Protocolos

**EN:**
- VLANs for network segmentation.
- Trunk links using IEEE 802.1Q with DTP.
- Spanning Tree Protocol (STP) for Layer 2 loop prevention.
- HSRP for gateway redundancy.
- DHCP for dynamic IP address allocation.
- Static routing and default routes.
- NAT for internet access simulation.

**PT-BR:**
- VLANs para segmentação de rede.
- Links trunk utilizando IEEE 802.1Q com DTP.
- Spanning Tree Protocol (STP) para prevenção de loops na camada 2.
- HSRP para redundância de gateway.
- DHCP para distribuição dinâmica de endereços IP.
- Roteamento estático e rotas padrão.
- NAT para simulação de acesso à internet.

Internet Simulation | Simulação de Internet

**EN:**  
Internet access is simulated using a loopback address (8.8.8.8), representing an external network.  
This approach allows testing routing, NAT behavior, and connectivity without relying on a real ISP connection.

**PT-BR:**  
O acesso à internet é simulado utilizando um endereço loopback (8.8.8.8), representando uma rede externa.  
Essa abordagem permite testar o roteamento, o comportamento do NAT e a conectividade sem depender de um provedor real.
